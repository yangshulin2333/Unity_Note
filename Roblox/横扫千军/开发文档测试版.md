
# 《横扫千军：Roblox 版》游戏开发设计文档 (GDD) — 完整升级版

---

## 1. 游戏概述与核心架构

- **游戏类型**: 3D 动作策略大亨 (Action-Strategy Tycoon + RPG)
    
- **核心循环**:  
    资源产出 (Tycoon) ➔ 抽卡/招募武将 (Gacha) ➔ 组建多维度阵容 (武将 + 战马 + 小兵) ➔ 挑战副本/抢夺领地 ➔ 获取养成材料反哺主城。
    
- **技术架构建议**:
    
    - 客户端-服务端分离架构 (Client-Server)
        
    - 玩家数据存储使用 `ProfileService` 或 `DataStoreService`
        
    - 养成模块采用面向对象 OOP 风格的 `ModuleScript`
        
    - 网络交互严格服务端结算，防作弊设计
        

---

## 2. 核心养成系统

### 2.1 武将系统 (Hero System)

武将为玩家核心战力，表现为跟随玩家或 NPC 自动寻敌的单位。

- **基础维度**:
    
    - **等级 (Level)**: 消耗经验书升级，等级上限受主公等级限制
        
    - **星级 (Star Rank)**: 收集碎片升星，解锁新技能
        
    - **品质进阶 (Quality)**: 绿 → 蓝 → 紫 → 橙 → 金 → 红 → 耀金，伴随武将材质变化或粒子特效升级
        
- **高阶系统**:
    
    - **天资图谱 (Talent Tree)**: 分支技能/属性加成
        
    - **羁绊 (Bonds)**: 特定武将组合上阵触发团队属性增益
        
- **数据结构示例 (Luau)**:
    

```lua
local HeroData = {
    Id = "GuanYu_001",
    Level = 1,
    Star = 1,
    Quality = "Purple",
    Equips = {}, -- 装备槽位
    Horse = nil, -- 战马槽位
    TroopType = "Crossbow"
}
```

---

### 2.2 装备系统 (Equipment System)

装备提升武将属性和技能效果。

- **获取与品质**: 白/绿/蓝/紫/橙/红六阶，通过副本掉落、祭祀、活动获得
    
- **强化与精炼**: 强化 +20，紫色及以上可精炼增加百分比属性
    
- **套装羁绊**: 集齐套装触发属性加成
    
- **宝石镶嵌**: 每件装备 4 个孔位，可镶嵌宝石升级属性
    

---

### 2.3 战马系统 (Mount/Companion)

战马作为武将核心坐骑，提供加成和技能。

- **获取**: 普通召唤/高级召唤 (Robux 或游戏货币)
    
- **养成线**: 消耗草料升级，升星解锁技能
    
- **核心战马示例**:
    
    - 赤兔: 普攻+攻速
        
    - 的卢: 控制技能回怒
        
    - 绝影: 死亡后免伤
        

---

### 2.4 部曲系统 (Minion/Squad)

武将带领的小兵，形成冲锋视觉效果。

- **招募与上阵限制**: 每位武将最多招募对应小兵，消耗募兵令
    
- **进阶路径**: 小兵训练等级、阶级、种类
    
- **特殊兵种**: 元戎弩兵/无当飞军/突豹骑，各具增益效果
    

---

## 3. 主城与经济系统 (Tycoon & Economy)

### 3.1 主城建筑

- **核心建筑**: 主殿、兵营、马场
    
- **资源建筑**: 农田、矿场、民居、酒馆 → 自动产出资源
    
- **主公技能**: 聚气释放，全局大招按钮
    
- **科技研发**: 投入资源提升全局属性倍率
    

### 3.2 经济模型

四种核心货币：

1. 铜币 (Coins): 基础货币，生产与掉落
    
2. 粮草 (Food): 副本出征/体力消耗
    
3. 铁矿 (Iron): 装备强化与建筑升级
    
4. 元宝 (Premium/Robux): 充值货币，用于抽高级战马、购买加速
    

---

## 4. 战斗与副本系统

- **副本实例化**: 每次挑战生成独立副本
    
- **单位控制**:
    
    - 玩家操控武将/战马
        
    - AI 控制小兵与敌方
        
    - PathfindingService + Humanoid 控制寻路与攻击
        
- **战斗结算**: 所有数值计算在服务端完成
    
- **奖励分发**: 副本掉落、经验、装备返还主城
    

---

## 5. 技术实现与模块化

|模块|功能|实现方式|
|---|---|---|
|武将系统|属性/技能/升星/进阶|ModuleScript + ServerScript|
|装备系统|装备管理/强化/套装|ModuleScript + RemoteEvent|
|战马系统|坐骑技能/养成/升星|ModuleScript + ServerScript|
|部曲系统|小兵招募/进阶/技能|ModuleScript + Pathfinding|
|主城|建筑/资源产出|Tycoon + ModuleScript|
|UI|状态栏/技能按钮/副本界面|ScreenGui + LocalScript|
|FX|粒子/音效/技能特效|ParticleEmitter + Sound|
|数据存储|玩家数据/战斗数据|ProfileService / DataStoreService|

## 6. 开发阶段与里程碑

1. **Alpha (1-30级)**
    
    - Tycoon 主城搭建
        
    - 基础战斗系统
        
    - 武将升级/装备系统实现
        
2. **Beta (31-60级)**
    
    - 战马系统上线
        
    - 装备精炼、科技树、武将升星
        
    - 副本生成逻辑完善
        
3. **Release (61+级)**
    
    - 红色/耀金武将特效
        
    - 部曲系统上线，多 NPC 战斗优化
        
    - 全面数值平衡与防作弊
        

---

## 7. 网络安全与优化

- 所有战斗计算、掉落和资源分发必须在服务端完成
    
- 客户端仅播放动画和更新 UI
    
- 大量 NPC 同屏时使用性能优化策略（分帧更新、远程卸载、LOD 简化模型）

## 9. UI/UX 设计建议

- 主城界面：Tycoon 风格资源产出 + 可点击建筑升级
    
- 战斗界面：操作武将、释放技能按钮、部曲 AI 自动战斗
    
- 副本界面：战斗开始/结束奖励结算弹窗
    
- 全局技能/大招按钮：屏幕右下角，可绑定主公技能
