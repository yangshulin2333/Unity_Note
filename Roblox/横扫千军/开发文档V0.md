# 1. 项目定位

## 1.1 游戏名称
**《横扫千军：Roblox 版》**

## 1.2 游戏类型
- 3D 动作策略养成模拟器
- Tycoon + RPG + Companion + Dungeon
- 重度成长 + 数值驱动 + 长线留存

## 1.3 核心卖点
1. **主城建设**：玩家拥有自己的主城/基地
2. **武将养成**：招募、升级、升星、进阶、羁绊
3. **战马与部曲**：随从化、可视化、战斗参与
4. **副本推图**：实例化战斗地图，刷装备和材料
5. **多系统联动**：主公、科技、建筑、装备互相加成
6. **Roblox 易理解外壳**：用 Tycoon + 宠物 + 副本的方式包装传统卡牌养成

# 2. 核心玩法循环

完整循环如下：

**进入主城 → 收资源 → 升建筑 → 招募武将/战马 → 强化英雄 → 配装 → 带队进副本 → 获得装备/碎片/材料 → 回主城继续养成 → 提升战力 → 挑战更高难度内容**

进一步拆成 Roblox 玩家能理解的循环：

### 日常循环
1. 登录
2. 领取离线收益/资源建筑收益
3. 升级主殿、兵营、马场
4. 抽武将/抽马
5. 给主力武将升级、升星、换装备
6. 进入副本或野外挑战
7. 获取掉落
8. 回到主城继续强化
9. 参加限时活动/排行榜/PVP

---

## 10.2 Plot 分配系统

### 实现目标
玩家进入服务器后：
1. 自动分配一个空地
2. 该地块归属该玩家
3. 玩家离开后释放地块
4. 建筑状态根据存档恢复

### 实操步骤
#### 1）在 Workspace 建立地块
- 新建 Folder：`Plots`
- 在里面创建 4~8 个地块模型
- 每个地块命名：
  - `Plot_1`
  - `Plot_2`
  - `Plot_3`
  - `Plot_4`

#### 2）每个 Plot 里放什么
```text
Plot_1
├── BaseSpawn
├── BuildNodes
├── CollectPoints
└── Decorations
```

#### 3）BuildNodes 用法
在地块里放一些空 Part，作为建筑位置点：
- `MainHallNode`
- `BarracksNode`
- `StableNode`
- `FarmNode`
- `MineNode`
- `HouseNode`
- `TavernNode`

#### 4）服务端分配逻辑
由 `BaseService` 负责：
- 玩家加入时扫描空 Plot
- 找一个未占用 Plot
- 记录 `plot.OwnerUserId = player.UserId`
- 传给客户端显示“你的主城”

---

## 10.3 建筑升级逻辑

每种建筑有：
- 当前等级
- 最大等级
- 升级消耗
- 解锁条件
- 产出效率

### 建议配置表
在 `ReplicatedStorage/Shared/Config/BuildingConfig` 建 ModuleScript：

```lua
local BuildingConfig = {
    MainHall = {
        MaxLevel = 20,
        Levels = {
            [1] = {UpgradeCost = {Coins = 100}, BuildTime = 0, Unlock = {}},
            [2] = {UpgradeCost = {Coins = 200}, BuildTime = 10, Unlock = {}},
        }
    },
    Farm = {
        MaxLevel = 20,
        Levels = {
            [1] = {OutputPerMinute = 10, Capacity = 100, UpgradeCost = {Coins = 50}},
            [2] = {OutputPerMinute = 20, Capacity = 200, UpgradeCost = {Coins = 120}},
        }
    }
}
return BuildingConfig
```

---

## 10.4 资源建筑产出

### 建议机制
农田、矿场、民居、酒馆按时间产出：
- 农田：粮草
- 矿场：铁矿
- 民居：铜币
- 酒馆：招募材料/碎片概率资源

### 实现方案
不要每秒给资源。  
推荐做成“**懒结算**”：

玩家打开界面或点击收取时：
1. 读取上次收取时间
2. 根据时间差算产出
3. 限制最大储量
4. 发放资源
5. 更新时间戳

这样性能最好。

## 11.3 武将获取
来源：
1. 酒馆招募/Gacha
2. 活动奖励
3. 碎片合成
4. 新手任务

---

## 11.4 武将升级
### 规则
- 消耗经验书 + 铜币
- 等级不能超过主公等级

### 公式建议
```text
升级所需经验 = 基础值 × 等级系数
```

### 服务器流程
1. 客户端点击“升级”
2. 发送武将实例ID + 使用经验书数量
3. 服务端校验：
   - 玩家是否拥有该武将
   - 材料是否足够
   - 等级是否达到主公上限
4. 服务端扣除资源
5. 服务端更新武将等级
6. 返回新数据给客户端

---

## 11.5 武将升星
### 规则
- 消耗同名碎片 + 升星石
- 最高 10 星
- 每升 1 星提升基础属性倍率
- 某些星级解锁技能增强

### 建议星级倍率
- 1星：100%
- 2星：115%
- 3星：130%
- 4星：150%
- 5星：175%
- 6星：205%
- 7星：240%
- 8星：280%
- 9星：325%
- 10星：380%

---

## 11.6 品质进阶
### 规则
绿 → 蓝 → 紫 → 橙 → 金 → 红 → 耀金

进阶消耗：
- 进阶丹
- 同名碎片
- 特殊材料（高阶）

效果：
- 提升成长率
- 解锁被动/觉醒
- 模型外观升级

---

## 11.7 技能系统
每个武将四类技能：
1. 主动技能
2. 被动1
3. 被动2
4. 觉醒技能

```lua
        Effects = {
            {Type = "Damage", Value = 2.0}
        }
    }
}
return SkillConfig
```

## 11.8 羁绊系统

### 示例
- 五虎上将：攻击+20%

### 实现逻辑
服务端计算当前阵容：
1. 读取玩家上阵武将列表
2. 扫描是否满足 BondConfig
3. 汇总加成
4. 注入战斗属性计算

# 24. 逐步实操开发流程

下面我开始写“你真的该怎么做”。

---

## 第 1 步：搭建基础目录

### 操作
在 Studio 中依次创建前面提到的所有 Folder 和 Script。

### 结果
你应当拥有：
- ReplicatedStorage/Shared
- ServerScriptService/Services
- ServerStorage/Data
- StarterGui 的多个 ScreenGui
- StarterPlayerScripts/Controllers

---

## 第 2 步：接入存档系统

### 操作
1. 下载/复制 `ProfileService`
2. 放入 `ServerScriptService/Packages` 或 `ServerStorage/Packages`
3. 建立 `DefaultProfile` ModuleScript
4. 写 `DataService`

### DataService 要做的事
- 玩家加入时加载存档
- 没有存档就使用默认数据
- 玩家离开时保存
- 提供 `GetProfile(player)` 方法

---

## 第 3 步：建立 Remote 通信

### 在 ReplicatedStorage/Shared/Remotes 建：
- `RequestUpgradeBuilding`
- `RequestCollectResource`
- `RequestSummonHero`
- `RequestUpgradeHero`
- `RequestEnterDungeon`
- `RequestEquipItem`
- `RequestUpgradeEquipment`
- `RequestSummonMount`

建议每个 Remote 单独命名明确。

---

## 第 4 步：实现玩家进入流程

### 逻辑
1. 玩家加入
2. DataService 读取档案
3. PlayerService 创建 session 数据
4. BaseService 分配 Plot
5. 把玩家传送到自己的基地出生点
6. 客户端打开主界面 HUD

---

## 第 5 步：制作 Plot 地块

### 操作
1. 在 Workspace 建 4 个 Plot 模型
2. 每个 Plot 做边界
3. 每个 Plot 放建筑节点
4. 放一个 SpawnLocation 或 Part 命名 `BaseSpawn`

### 注意
- Plot 要彼此距离足够远
- 避免玩家误入其他人基地
- 可设置透明墙或权限区

---

## 第 6 步：建立建筑升级系统

### 操作细分
#### A. 配置 BuildingConfig
定义每级升级成本与解锁

#### B. 在每个建筑节点放交互触发器
可用：
- ProximityPrompt
- ClickDetector
- 触发 UI 按钮

#### C. 服务端处理升级
`BaseService:UpgradeBuilding(player, buildingType)`

流程：
1. 读档
2. 检查当前等级
3. 检查主殿限制
4. 检查资源是否足够
5. 扣资源
6. 提升等级
7. 刷新建筑外观

#### D. 建筑外观变化
做法：
- 每级对应一个模型
- 升级后替换模型
或者
- 同一模型中显示不同部件

**建议首版用“替换模型”最直观。**

---

## 第 7 步：实现资源建筑收菜

### 操作
1. 每种资源建筑记录：
   - 等级
   - 上次收取时间
2. 玩家点击“收取”
3. 服务端按时间差结算
4. 发放资源
5. 更新上次收取时间

### UI
在建筑头顶放 BillboardGui：
- 当前储量
- 收取按钮提示

---

## 第 8 步：实现酒馆招募武将

### 操作
1. 创建 `HeroConfig`
2. 建立招募池 `GachaConfig`
3. 创建 `HeroService`

### 招募逻辑
- 普通招募：消耗 Coins
- 高级招募：消耗 Gems / 券

### 服务端抽卡流程
1. 客户端点抽卡
2. 服务端校验货币
3. 按概率表抽结果
4. 如果重复，转化为碎片或叠加
5. 写入玩家背包
6. 返回结果给客户端播放演出

---

## 第 9 步：制作武将 UI 界面

界面至少包含：
- 武将列表
- 武将头像
- 名称
- 品质
- 等级
- 星级
- 战力
- 升级按钮
- 上阵按钮
- 装备槽

---

## 第 10 步：实现武将跟随玩家

### 简单做法
1. 玩家上阵一个武将
2. 在玩家身后生成武将 NPC 模型
3. 武将始终尝试靠近玩家的偏移点

### 技术建议
- 用 `Humanoid:MoveTo()`
- 或使用更稳定的跟随逻辑
- 更新频率不要太高

---

## 第 11 步：实现武将自动战斗

### 基础逻辑
1. 扫描范围内敌人
2. 找最近敌人
3. 进入攻击范围
4. 普攻
5. CD 好后放技能

### 服务端处理
由 `CombatService` 统一处理伤害。

---

## 第 12 步：实现第一张副本

### 操作
1. 在 ServerStorage/Templates/Dungeons 建一个模型 `Dungeon_1`
2. 里面放：
   - 玩家出生点
   - 敌人出生点
   - Boss出生点
   - 终点门

3. 写 `DungeonService`
4. 进入副本时克隆地图到 Workspace/ActiveDungeons

---

## 第 13 步：制作敌人 AI

### 敌人结构
- Model
- Humanoid
- HumanoidRootPart
- EnemyAI Script 或通过服务端模块控制

### AI 第一版
- 锁最近目标
- MoveTo 靠近
- 进入范围攻击
- 死亡掉落由服务端统一处理

---

## 第 14 步：副本奖励结算

### 结算内容
- 铜币
- 经验
- 装备
- 材料
- 武将碎片
- 募兵令
- 草料

### 结算流程
1. 最后一波敌人死亡 / Boss死亡
2. 服务端确认副本成功
3. 根据掉落表生成奖励
4. 写入玩家数据
5. 客户端显示结算面板
6. 传送回主城

---

## 第 15 步：实现装备掉落与背包

### 操作
1. 创建 `EquipmentConfig`
2. 创建 `DropTableConfig`
3. 打副本时随机生成装备实例
4. 存入 `EquipmentInventory`

### 背包 UI
- 按品质排序
- 按部位筛选
- 一键穿戴
- 强化入口

---

## 第 16 步：实现装备穿戴与强化

### 穿戴流程
1. 选武将
2. 选装备
3. 服务端校验装备部位
4. 如果槽位已有装备，先卸下旧装备
5. 写入武将装备槽

# 26. 首个可玩版本（MVP）目标

如果你想真的把项目做出来，**千万不要一开始做全套系统**。  
MVP 版本只做这些：

## MVP 必做内容
1. 玩家存档
2. Plot 主城
3. 主殿/农田/矿场/酒馆/兵营 五个建筑
4. 铜币/粮草/铁矿 三种资源
5. 武将招募
6. 一个主力武将上阵
7. 武将升级
8. 一个副本
9. 敌人 AI
10. 装备掉落
11. 装备穿戴强化
12. 结算界面

## MVP 不做内容
- 羁绊
- 耀金血脉
- 将星录
- 宝石淬炼
- 多人PVP
- 官职争霸
- 大规模国战
