# 《横扫千军：Roblox 版》游戏开发设计文档 (GDD)

## 1. 游戏概述与核心架构

- **游戏类型**: 3D 动作策略大亨 (Action-Strategy Tycoon + RPG)
    
- **核心循环**: 资源产出 (Tycoon) ➔ 抽卡/招募武将 (Gacha) ➔ 组建多维度阵容 (武将+战马+小兵) ➔ 挑战副本/抢夺领地 ➔ 获取养成材料反哺主城。
    
- **技术架构建议**: 采用客户端-服务端 (Client-Server) 分离架构。推荐使用 `ProfileService` 进行玩家复杂数据的存储，使用面向对象 (OOP) 的 `ModuleScript` 来管理各类养成模块。

## 2. 核心养成系统设计 (Luau 数据层映射)

### 2.1 武将系统 (核心战力/Hero System)

武将是玩家的“英雄实体”，在 Roblox 中可以表现为跟随玩家的 NPC，自动寻敌战斗。

- **基础维度**:
    
    - **等级 (Level)**: 消耗经验书升级，等级上限受玩家（主公）等级限制 。
        
    - **星级 (Star Rank)**: 收集同名碎片升星（最高10星），解锁新技能效果 。
        
    - **品质进阶 (Quality)**: 按照 `绿 ➔ 蓝 ➔ 紫 ➔ 橙 ➔ 金 ➔ 红 ➔ 耀金` 的线性路径进阶，消耗进阶丹 。在 Roblox 中，进阶可以伴随武将模型的材质 (Material) 发光效果或粒子特效 (ParticleEmitter) 升级。
        
- **高阶系统**:
    
    - **天资图谱**: 类似于天赋树，包含多条分支（如主公聚气速度、核心属性），资质达到180+需突破壁障 。
        
    - **羁绊 (Bonds)**: 特定武将组合上阵触发，提升如攻击+20%、防御+15%的团队属性 。

### 2.2 装备锻造系统 (Equipment System)

装备用于强化武将的基础数值。

- **品质与获取**: 包含白、绿、蓝、紫、橙、红六个等级 。主要通过普通/精英关卡掉落、兵器冢副本以及每日祭祀获取 。
    
- **强化与套装**:
    
    - **强化**: 消耗强化石最高至+20级 。
        
    - **精炼**: 紫色以上装备解锁，消耗精炼石提供百分比属性加成 。
        
    - **套装羁绊**: 集齐2/4/6件同套装触发强力羁绊（如暴击率、免伤） 。
        
- **宝石镶嵌**:
    
    - 每件装备4个独立孔位，需镶嵌不同类型宝石（共12种属性） 。
        
    - 宝石通过吞噬升级，9级以上解锁“淬炼” 。
        

### 2.3 战马系统 (Pet/Mount System)

在 Roblox 玩家群体中，开蛋/抽宠物是非常核心的付费与留存点。战马可直接转化为“坐骑/宠物”系统。

- **获取机制 (Gacha)**: 马场普通召唤（消耗铜币，小概率出紫）与高级召唤（消耗元宝/Robux，必出紫以上） 。
    
- **养成线**: 消耗草料升级，消耗同名马魂最高升至5星 。3星解锁基础技能，5星技能最强 。
    
- **顶级战马配置 (核心效果)**:
    
    - **赤兔 (输出核心)**: 普攻40%概率+30%攻速（持续3秒） 。
        
    - **的卢 (辅助/控制)**: 释放控制效果时全体回怒（Mana恢复） 。
        
    - **绝影 (前排坦克)**: 死亡后免疫伤害一段时间，提升生存时间 。
        

### 2.4 部曲系统 (Minion/Squad System)

主将带领的小兵，在 3D 战场上可以形成“军团冲锋”的视觉震撼效果。

- **招募与限制**: 同一时间仅1名上阵武将可招募小兵，消耗通关或屯田获取的募兵令 。
    
- **进阶路径**: 消耗兵书提升训练等级（每10级进阶），消耗军械可升至最高6阶，2-5阶分化小兵种 。
    
- **传说兵种**:
    
    - **元戎弩兵**: 增加统帅技能伤害（适合技能爆发武将） 。
        
    - **无当飞军**: 提供攻速加成（适合持续输出武将） 。
        
    - **突豹骑**: 大幅提升暴击率（适合骑兵系收割） 。

## 3. 基地与内政系统 (Tycoon & Economy)

### 3.1 主城建筑 (Base Building)

玩家的出生点是一个专属的 Tycoon 区域。

- **核心建筑**: 主殿、兵营、马场（决定基础战力上限与解锁功能） 。
    
- **资源建筑**: 农田、矿场、民居、酒馆（随时间自动生成资源，类似 Tycoon 的 Dropper） 。
    
- **主公技能**: 主殿解锁主公技，需聚气释放，分为伤害、辅助、聚气三类 。这可以做成玩家屏幕右下角的全局大招按钮。
    
- **科技研发**: 在兵营和建筑科技节点中投入资源，获得全局的数值倍率加成 (Multiplier) 。
    

### 3.2 经济模型 (Leaderstats & Monetization)

系统内的四种核心货币必须严格控制产销平衡：

1. **铜币 (Coins)**: 基础货币，用于武将招募、技能升级与进阶 。通过民居产出和打怪掉落。
    
2. **粮草 (Food)**: 门票资源，主要用于国战出征/进入副本的体力消耗 。
    
3. **铁矿 (Iron)**: 强化资源，用于高级装备打造和建筑升级 。
    
4. **元宝 (Premium/Robux)**: 充值货币，用于限时活动、抽高级战马、购买加速道具 (Skip Wait Time) 。

## 4. 发展规划与开发里程碑建议

按照原文档的养成联动策略 ，Roblox 版本的开发也可以分为三个阶段上线：

- **Alpha 版 (对应前期 1-30级)**
    
    - **核心开发**: 跑通 Tycoon 主城搭建逻辑。
        
    - **战斗实现**: 完成武将基础的移动、攻击脚本，实现基础的等级与装备强化功能。
        
- **Beta 版 (对应中期 31-60级)**
    
    - **核心开发**: 引入战马系统 (Gacha UI) 抽奖逻辑。
        
    - **系统深化**: 装备进阶精炼、科技树研发模块实装，并加入武将升星逻辑。
        
- **Release 版 (对应后期 61级+)**
    
    - **核心开发**: 红色/耀金武将材质特效升级 。
        
    - **大战场**: 实装“部曲”系统，优化多 NPC 寻路 (Pathfinding) 性能，确保服务器不会因为同屏大量兵卒而产生严重卡顿。
        

在 Roblox 中实现这种重度数值游戏，最核心的挑战在于 **网络同步与防作弊**。建议所有的战斗结算（伤害计算、掉落概率、资源增加）必须在 `ServerScriptService` 中的服务端脚本完成，客户端（`StarterPlayerScripts`）仅负责接收 `RemoteEvent` 来播放动画、特效和更新 UI。
