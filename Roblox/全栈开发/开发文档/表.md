# Luau 表（Table）核心知识点详解

## 一、表的本质与基础定义

### 1. 核心特性

- 表是 **引用类型**（不是值类型），赋值 / 传参时传递的是引用，不是副本
- 可以存储任意类型的值：`nil`、布尔值、数字、字符串、函数、其他表
- 用 `{}` 构造表，支持两种核心形态：**数组（有序列表）** 和 **字典（键值对）**

### 2. 基础语法


```lua
-- 1. 空表
local t = {}
print(t) -- 输出表的内存地址（如 table: 0x...）

-- 2. 数组形态（有序列表，数字索引）
local testArray = {"一个字符串", 3.14159, true}

-- 3. 字典形态（键值对，任意类型索引）
local testDictionary = {
    fruitName = "柠檬",
    sour = true
}
```

---

## 二、数组（Array）：有序列表

### 1. 关键特性：1-based 索引（从 1 开始）

⚠️ 这是 Luau 最容易踩的坑！和 C#/C++ 等语言的 0-based 完全不同：

- 数组第一个元素索引是 `[1]`，不是 `[0]`
- 索引必须是连续的正整数，否则会被视为字典键

### 2. 数组操作全解

#### （1）读取元素


```lua
local testArray = {"一个字符串", 3.14159, true}
print(testArray[1]) -- 输出：一个字符串
print(testArray[2]) -- 输出：3.14159
print(testArray[3]) -- 输出：true
```

#### （2）写入 / 修改元素


```lua
local testArray = {"一个字符串", 3.14159, true}
-- 修改已有索引
testArray[2] = 12345
-- 新增元素（自动扩展数组）
testArray[4] = "新字符串"

print(testArray[2]) -- 输出：12345
print(testArray[4]) -- 输出：新字符串
```

#### （3）迭代数组（两种常用方式）

```lua
local testArray = {"一个字符串", 3.14159, true, "新字符串"}

-- 方式1：泛型for循环（推荐，自动遍历所有有效索引）
for index, value in testArray do
    print(index, value)
end

-- 方式2：数字for循环（用#获取数组长度，适合需要索引的场景）
for index = 1, #testArray do
    print(index, testArray[index])
end
```

> 补充：`#` 是**长度运算符**，仅对「连续数字索引的数组」生效，字典用它会得到不可预期的结果。
> Luau 遍历：只和**顺序**有关，和**变量名无关**

#### （4）插入元素

两种常用方法：

```luau
local testArray = {"一个字符串", 3.14159}

-- 方法1：table.insert（推荐，支持指定位置插入）
-- 末尾插入
table.insert(testArray, "新字符串")
-- 指定位置插入（第2位，后续元素后移）
table.insert(testArray, 2, "新项目#2")

-- 方法2：#语法（仅末尾插入，简洁）
testArray[#testArray + 1] = "另一个新字符串"
```

#### （5）移除元素

```luau
local testArray = {"第一个项目", "下一个项目", "最后一个项目"}
-- 移除第2位元素，后续元素自动前移
table.remove(testArray, 2)

print(testArray[1]) -- 第一个项目
print(testArray[2]) -- 最后一个项目
-- table.remove(testArray) -- 直接删掉数组最后一个元素，不用写下标
```

## 三、字典（Dictionary）：键值对集合

### 1. 核心特性

- 键（Key）可以是**任意非 nil 类型**：数字、字符串、对象、函数
- 字符串键有两种写法：`["key"]` 或 `.key`（语法糖）
- 无序存储，迭代顺序不固定

### 2. 字典操作全解

#### （1）定义与读取

```luau
local testDictionary = {
    fruitName = "柠檬", -- 字符串键简写
    sour = true,
    [100] = "数字键"    -- 非字符串键必须用[]
}

-- 读取：两种写法等价
print(testDictionary["fruitName"]) -- 柠檬
print(testDictionary.fruitName)    -- 柠檬（字符串键专属语法糖）
print(testDictionary[100])         -- 数字键
--Luau 表取值 只有 2 种
--表["键名"] → 万能，任何键都能用
--表.键名 → 只能给字符串键用（语法糖）
```
注意：

Luau 字典里：只有**字符串键**可以省略 []，其他所有键（数字、变量、表达式）必须加 []
#### （2）写入 / 修改元素

lua

```
local testDictionary = {
    fruitName = "柠檬",
    sour = true
}

-- 修改已有键
testDictionary["fruitName"] = "樱桃"
testDictionary.sour = false -- 语法糖写法

-- 新增键值对
testDictionary.fruitCount = 10 -- 直接新增

print(testDictionary.fruitName)  -- 樱桃
print(testDictionary.sour)       -- false
print(testDictionary.fruitCount) -- 10
```

#### （3）迭代字典

只能用**泛型 for 循环**，不能用数字 for：

```luau
local testDictionary = {
    fruitName = "柠檬",
    fruitColor = "黄色",
    sour = true
}

for key, value in testDictionary do
    print(key, value)
end
-- 输出（顺序不固定）：
-- fruitName 柠檬
-- fruitColor 黄色
-- sour true
```

#### （4）移除键值对
字典删除 = 把**键赋值为 nil**
```
local t = { name = "小明", age = 20, [100] = "数字键" } 
-- 删除字典键 
t.name = nil -- 删除 
name t.age = nil -- 删除 
age t[100] = nil -- 删除数字键 100
```

## 四、表的引用特性（Roblox 开发高频坑）

### 1. 引用传递 vs 值传递

✅ 值类型：nil、boolean、number、string → 复制数据，互不影响
✅ 引用类型：table、function、userdata、thread → 复制地址，共享数据

表是**引用类型**，赋值时不会创建副本，而是指向同一个内存地址：

lua

```
local originalArray = {10, 20}
local arrayReference = originalArray -- 传递引用，不是副本

-- 修改原表，引用表也会变
originalArray[1] = 100
originalArray[2] = 200

print(originalArray[1], originalArray[2]) -- 100 200
print(arrayReference[1], arrayReference[2]) -- 100 200（同步变化）
```

### 2. 表的深拷贝（解决引用问题）

要在没有任何嵌套表的情况下复制表，Luau 提供了 table.clone() 方法。

如果需要完全独立的副本，必须手动实现深拷贝：
```luau
local function deepClone(original)
    local clone = {}
    for key, value in original do
        if type(value) == "table" then
            -- 递归拷贝嵌套表
            clone[key] = deepClone(value)
        else
            clone[key] = value
        end
    end
    return clone
end

-- 使用
local original = {a = 1, b = {c = 2}}
local copy = deepClone(original)
copy.b.c = 100
print(original.b.c) -- 2（原表不受影响）
print(copy.b.c)     -- 100
```

## 五、表的混合形态（数组 + 字典）

Luau 表支持**数组和字典共存**，这是它的灵活性所在：

lua

```
local mixedTable = {
    -- 数组部分（连续数字索引）
    "玩家1", "玩家2", "玩家3",
    -- 字典部分（字符串键）
    maxPlayers = 10,
    gameStatus = "进行中"
}

-- 数组部分用#、ipairs遍历
print(#mixedTable) -- 3（数组长度）
for i = 1, #mixedTable do
    print(mixedTable[i]) -- 玩家1、玩家2、玩家3
end

-- 字典部分用pairs遍历
for key, value in mixedTable do
    print(key, value) -- 会同时输出数组和字典的所有键值对
end
```

> 注意：混合表用 `#` 只会计算**连续数字索引的长度**，字典部分不会被计入。

## 六、table 库常用函数（Roblox 开发必备）

表格

|函数|作用|示例|
|---|---|---|
|`table.insert(t, pos, value)`|插入元素，pos 可选（默认末尾）|`table.insert(arr, 2, "新元素")`|
|`table.remove(t, pos)`|移除元素，pos 可选（默认末尾）|`table.remove(arr, 1)`|
|`table.concat(t, sep)`|数组转字符串，sep 为分隔符|`table.concat(arr, ",")`|
|`table.sort(t, comp)`|数组排序，comp 为自定义比较函数|`table.sort(arr, function(a,b) return a > b end)`|
|`table.move(t, from, to, dest)`|批量移动数组元素|`table.move(arr, 1, 3, 5)`|

## 七、Luau 表 vs C# 对应结构（入职速记）

表格

|Luau 表形态|C# 对应类型|核心区别|
|---|---|---|
|数组（连续数字索引）|`List<T>` / 数组|Luau 1-based，C# 0-based；Luau 动态扩容|
|字典（键值对）|`Dictionary<TKey, TValue>`|Luau 键类型更灵活，支持混合键|
|混合表|无直接对应|C# 无法在一个集合中同时存数组和字典|
|引用特性|引用类型（类）|两者都是引用传递，逻辑一致|

## 九、避坑指南（入职必看）

1. **数组索引从 1 开始**：绝对不要用 `[0]` 访问第一个元素
2. **`#` 长度运算符的坑**：仅对连续数字索引的数组生效，字典 / 稀疏数组用它会出错
3. **引用传递的坑**：修改表会影响所有引用它的变量，需要副本必须深拷贝
4. **nil 不能作为键**：表的键不能是 `nil`，值可以是 `nil`（设为 nil 会删除该键）
5. **迭代顺序**：字典迭代顺序不固定，不要依赖顺序做逻辑
