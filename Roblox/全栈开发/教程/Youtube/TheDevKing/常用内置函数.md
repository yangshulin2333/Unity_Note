这个视频讲解了 Roblox 开发中几个非常常用且核心的**内置函数**。作为全栈开发，这些是你每天写脚本都会用到的“工具”。

以下是重点总结：

### 1. `GetChildren()`：获取所有子集 

- **作用**：获取一个对象下面**直接**包含的所有子对象，并把它们放进一个**表（Table）**里。
    
- **应用场景**：如果你想一次性删除文件夹里的所有零件，或者给所有灯光变色，就用它配合 `for` 循环 。
    ```Lua
    local folder = workspace.Parts
    for i, v in pairs(folder:GetChildren()) do
        v:Destroy() -- 循环删除所有子对象
    end
    ```
    

### 2. `WaitForChild()`：等一下子集

- **作用**：脚本运行速度极快，有时游戏还没加载完，脚本就开始找模型，导致报错。这个函数会让脚本**等**那个模型出现后再继续运行。
    
- **重要性**：**开发必记！** 在 `LocalScript` 里获取玩家角色或 UI 时，几乎必须用它，否则经常会报 `Infinite yield possible`（可能无限等待）或找不到对象的错。
    

### 3. `GetService()`：获取服务 

- **作用**：获取 Roblox 的核心系统功能。 返回请求的类名的服务，如果它不存在则创建它。
    
- **常用服务**：
    
    - `ReplicatedStorage`：存放两端都能访问的资源 。
        
    - `TweenService`：做平滑动画。
        
    - `Players`：获取玩家信息。
        
- **专业习惯**：虽然可以直接用 `game.Players`，但专业写法是 `game:GetService("Players")`，这更安全、更符合规范 [[08:50](http://www.youtube.com/watch?v=w4k_WRg4da8&t=530)]。
    

### 4. `FindFirstChild()`：找找看有没有这个孩子 [[09:09](http://www.youtube.com/watch?v=w4k_WRg4da8&t=549)]

- **作用**：如果直接写 `part.Head` 而 Head 不存在，脚本会崩掉。用这个函数，如果没找到它会返回 `nil` 而不报错 [[11:04](http://www.youtube.com/watch?v=w4k_WRg4da8&t=664)]。
    
- **递归搜索**：它有一个可选参数 `true`。如果加上，它不仅会找直接的孩子，还会钻进子文件夹里找遍所有后代 [[11:25](http://www.youtube.com/watch?v=w4k_WRg4da8&t=685)]。
    

### 5. `Clone()`：克隆/复制 [[11:43](http://www.youtube.com/watch?v=w4k_WRg4da8&t=703)]

- **作用**：复制一个现有的对象 [[11:50](http://www.youtube.com/watch?v=w4k_WRg4da8&t=710)]。
    
- **注意点**：克隆出来的东西默认在内存里，是看不见的。你必须**手动设置它的 Parent（父级）**，它才会出现在游戏世界里 [[15:00](http://www.youtube.com/watch?v=w4k_WRg4da8&t=900)]。
    
    Lua
    
    ```
    local newPart = oldPart:Clone()
    newPart.Parent = workspace -- 这一步非常关键！
    ```
    

### 6. `Destroy()`：摧毁 [[15:33](http://www.youtube.com/watch?v=w4k_WRg4da8&t=933)]

- **作用**：将对象从游戏中永久移除并释放内存 [[15:44](http://www.youtube.com/watch?v=w4k_WRg4da8&t=944)]。
    

---

### 💡 针对你明天的入职建议：

这些函数中，**`WaitForChild`** 和 **`GetChildren`** 是你最先要练熟的。

比如你之前弄那个李洛克模型，如果你想在脚本里控制他，最好写成：

Lua

```
local lee = workspace:WaitForChild("StarterCharacter")
```

这样能保证不管玩家网速多慢，脚本都能稳稳地抓到模型。

**你现在在 Studio 里尝试用 `GetChildren()` 打印出你那个李洛克模型里所有零件的名字了吗？** 这能帮你快速理解“表”和“层级”的关系。
