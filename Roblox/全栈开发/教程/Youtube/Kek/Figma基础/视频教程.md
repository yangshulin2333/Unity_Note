# Roblox UI 导入知识点笔记

## 1. 视频核心主题

本视频讲解如何把在 **Figma / Photoshop / Affinity / Photopea** 等第三方设计软件中制作的 UI，正确导入到 **Roblox Studio** 中，并让它在不同设备分辨率下保持良好缩放和可交互性。

核心工具是作者自制插件 **Kekui / KUI**，用于加速 UI 的缩放、定位、约束、描边、圆角等处理。

---

## 2. UI Importing 是什么

**UI Importing** 指的是：

> 将第三方软件中设计好的 UI 元素，以图片、层级结构和 Roblox GUI 对象的形式，重新搭建到 Roblox Studio 中。

重点不是简单把整张图丢进 Studio，而是要：

- 拆分 UI 元素
- 保留可编辑文本
- 正确使用 Frame / ImageLabel / TextLabel / Button
- 设置缩放和约束
- 适配不同设备
- 让按钮、滚动区域、血条等真正可用

---

## 3. Exporting 与 Importing 的区别

### Exporting

从设计软件中导出素材。

不同软件流程不同：

- Photoshop
- Figma
- Affinity
- Photopea

### Importing

把导出的素材放进 Roblox Studio。

无论从哪个软件导出，只要最终得到 PNG 图片，导入 Roblox 的流程基本一致。

关键原则：

- 导出只是准备图片资源
- 导入才是在 Roblox 中重建 UI 结构
- 不要依赖一整张大图完成所有 UI
- 应该按模块拆分后重新搭建

---

## 4. 图片导出原则

## 使用 PNG，不推荐 JPG

原因：

- PNG 支持透明背景
- JPG 不支持透明，会出现黑色、灰色或实色背景
- PNG 更适合 UI、图标、按钮、边框、光效
- JPG 虽然体积小，但不适合需要透明度的 UI 元素

## 不要导出普通文字

普通文本应在 Roblox Studio 中用 `TextLabel` 或 `TextButton` 创建。

原因：

- 图片文字不可编辑
- 无法动态修改内容
- 不利于多语言、数值变化、玩家信息显示

例外情况：

- 字体或特效 Roblox Studio 无法还原
- 文字永远不会变化
- 需要完全保留设计效果

这种情况下可以把文字栅格化后作为图片导出。

## 图片尺寸控制在 1024px 以下

Roblox 会压缩或缩放过大的图片。

建议：

- 单张 UI 图片不要超过 `1024 x 1024`
- 大型 UI 应拆成多个部分
- 全屏 UI 不要直接导出为一整张大图

---

## 5. 账号与保存习惯

## 使用小号上传素材

作者建议不要用主账号上传 UI 图片、音效、贴图等资源。

原因：

- Roblox 审核可能误判
- 图片资源可能被 flag
- 主账号存在被限制或封禁风险

## 频繁保存

UI 导入过程较长，容易因为崩溃或文件损坏丢失进度。

建议：

- 经常 `Ctrl + S`
- 保存 Roblox 项目
- 必要时保留本地副本

## 6. 从不同软件导出 UI

## Photoshop

流程：

- 打开 PSD
- 找到需要导出的 UI 部分
- 隐藏文字图层
- 合并相关图层
- 导出为 PNG

注意：

- 不要把所有图层分得太碎
- 有纹理、渐变、阴影的部分可以合并为一张图
- 文本尽量保留到 Studio 中重建

## Figma

流程：

- 选择需要导出的 frame 或 group
- 隐藏文本图层
- 按模块导出 PNG
- 例如背景、卡片、按钮底图、图标分别导出

Figma 中要保持模块化：

- 主背景单独导出
- item/card 单独导出
- 图标单独导出
- 可重复结构不要全部导成一张死图

## Affinity

流程类似：

- 找到目标图层
- 隐藏文本
- 导出 PNG
- 有复杂效果的图层需要一起导出

## Photopea

可用但不推荐。

原因：

- 操作较繁琐
- 图层导出不够直观
- 容易误导出文字、蒙版、混合效果

## 8. 上传图片到 Roblox

通过 `Asset Manager` 上传：

1. 打开 `Asset Manager`
2. 选择 `Images`
3. 点击批量导入
4. 选择 PNG 文件
5. 上传后复制图片 Asset ID
6. 粘贴到 `ImageLabel` 或 `ImageButton` 的 Image 属性中

批量导入适合一次上传多个 UI 素材。

## 9. 常用 Roblox GUI 对象

## ScreenGui

2D UI 的主要容器，通常放在 `StarterGui` 下。

重要属性：

- `IgnoreGuiInset`：控制是否避开顶部系统栏
- `DisplayOrder`：类似 z-index，决定不同 ScreenGui 的显示层级
- `ResetOnSpawn`：玩家重生时 UI 是否重置

## BillboardGui

用于 3D 世界中的悬浮 UI。

常见用途：

- NPC 名字
- 血条
- 交互提示

重要属性：

- `AlwaysOnTop`
- `MaxDistance`
- `LightInfluence`
- `StudsOffset`

## SurfaceGui

附着在 3D 物体表面的 UI。

常见用途：

- 屏幕
- 看板
- 交互面板
- 贴在 Part 表面的 UI

## 10. UI 层级结构

导入 UI 时要重视 hierarchy。

基本结构示例：

```text
StarterGui
└── ScreenGui
    └── MainFrame
        ├── Header
        ├── ContentHolder
        ├── Button
        └── ScrollingFrame
```

注意：

- 父子关系会影响缩放、裁剪和显示
- 层级顺序会影响遮挡
- 需要合理设置 `ZIndex`
- 按钮不要被其他 Frame 挡住

---

## 11. 常用 GUI 元素

## Frame

最基础的 UI 容器。

用途：

- 放置其他 UI 元素
- 做背景
- 做布局容器
- 管理子元素

重要属性：

- `AnchorPoint`
- `Position`
- `Size`
- `BackgroundTransparency`
- `ZIndex`
- `AutomaticSize`

## TextLabel

用于显示文本。

常用属性：

- `FontFace`
- `TextSize`
- `TextScaled`
- `TextColor3`
- `TextStroke`
- `RichText`
- `TextXAlignment`
- `TextYAlignment`
- `LineHeight`

## ImageLabel

用于显示图片。

常用属性：

- `Image`
- `ImageColor3`
- `ImageTransparency`
- `ScaleType`
- `SliceCenter`
- `ImageRectOffset`
- `ImageRectSize`

## TextBox

可输入文本的 UI 元素。

适合：

- 搜索框
- 昵称输入
- 聊天输入
- 表单输入

重要属性：

- `PlaceholderText`
- `ClearTextOnFocus`
- `MultiLine`
- `TextEditable`

## TextButton / ImageButton

可点击按钮。

常用设置：

- 关闭默认 `AutoButtonColor`
- 使用自定义 hover / pressed 效果
- 图片按钮可设置 hover image 和 pressed image
- 必要时添加单独 hitbox

## 12. 重要 UI 组件

## UICorner

给 UI 添加圆角。

常用：

- 卡片
- 按钮
- 头像框
- 圆形图标

圆形通常设置：

```text
CornerRadius = 1, 0
```

## UIStroke

给 UI 添加描边。

可用于：

- Frame 边框
- TextLabel 文字描边
- 按钮高亮
- 卡片边缘

注意区分：

- `Contextual`：作用于内容，比如文字
- `Border`：作用于边界

## UIPadding

给容器内部增加边距。

但作者不太推荐滥用，因为可能影响动画和布局。

## UIScale

整体缩放 UI。

常用于：

- hover 放大
- 点击回弹
- 动画效果

建议配合居中的 `AnchorPoint` 使用。

## UIAspectRatioConstraint

保持宽高比例。

适合：

- 图标
- 卡片
- 按钮
- 头像
- 非全屏 UI 模块

不要滥用在整个全屏 HUD 上，否则会在不同屏幕比例下出现空白边。

## UIGradient

添加渐变，也可用来控制透明度。

用途：

- 背景渐变
- 光效
- 扫光动画
- 滚动区域淡出
- 血条/进度条遮罩

## UIListLayout

自动按水平或垂直方向排列子元素。

适合：

- 菜单
- 按钮组
- 列表
- 横向标签栏

## UIGridLayout

按网格排列子元素。

适合：

- 背包
- 商店
- 卡片列表
- 单位列表

## UIPageLayout

分页布局，可做简单滑动页面，但现在较少使用。

## UITableLayout

作者认为较过时，不推荐优先学习。

---

## 13. ScrollingFrame 重点

`ScrollingFrame` 是可滚动容器。

常见用途：

- 背包
- 商店
- 排行榜
- 任务列表
- 卡片墙

重要属性：

- `CanvasSize`
- `AutomaticCanvasSize`
- `ScrollBarThickness`
- `ScrollingDirection`
- `CanvasPosition`

建议：

- 配合 `UIGridLayout` 或 `UIListLayout`
- 使用 `AutomaticCanvasSize = Y` 让内容高度自动扩展
- 滚动条样式可通过 top/mid/bottom image 自定义

---

## 14. Grayscale 灰度导入

灰度导入指把图像导成白色或黑白，再在 Roblox Studio 中通过 `ImageColor3` 改色。

优点：

- 可动态换色
- 适合光效、描边、稀有度颜色、hover 效果
- 颜色更准确

如果原图本身带颜色，再用 `ImageColor3` 改色，结果会偏色。

适合灰度导入的元素：

- glow
- aura
- rarity border
- icon highlight
- button hover overlay

---

## 15. Hitbox 点击区域

有些 UI 看起来是按钮，但实际是 `ImageLabel`，不能点击。

解决方式：

- 在视觉元素上方放一个透明 `ImageButton`
- 调整大小覆盖可点击区域
- 设置透明背景
- 命名为 `Hitbox`

用途：

- 修正点击范围
- 避免按钮太大或太小
- 保留原来的视觉层级

---

## 16. 约束使用原则

不要给所有元素无脑加 `UIAspectRatioConstraint`。

错误做法：

- 给整个全屏 UI 加比例约束
- 给每个父子节点都加约束
- 导致移动端、宽屏、方屏出现空白或挤压

正确做法：

- 只给需要保持形状的局部元素加约束
- 例如图标、卡片、按钮、圆形头像
- 全屏 HUD 通常应自由适配屏幕比例

---

## 17. 滚动区域淡出效果

实现方式：

- 在 ScrollingFrame 上方放一个 Frame
- 设置合适大小和位置
- 添加 `UIGradient`
- 使用透明度渐变制造 fade 效果

用途：

- 让滚动内容边缘自然消失
- 避免元素突然被硬切

---

## 18. 导入型血条 / 进度条

不推荐直接横向缩放图片，因为会拉伸纹理。

更好的方式：

- 使用导入的完整条形图片
- 设置 `ScaleType = Fit`
- 添加 `UIGradient`
- 通过调整 gradient 的透明度位置控制显示范围

适合：

- HP bar
- boss bar
- energy bar
- loading bar

---

## 19. Slice / 9-Slice

Roblox 的 `Slice` 可以让图片拉伸时保留边角不变。

适合：

- 圆角按钮
- 长条框
- 弹窗背景
- 输入框
- 加载条背景

作用：

- 中间部分可拉伸
- 边缘和圆角不变形
- 比普通 Stretch 更适合 UI

---

## 20. 圆形进度条

作者演示了用多个 Frame、UIStroke、UICorner、旋转和遮罩组合出圆形进度条。

核心思路：

- 外层容器保持正方形
- 使用 `UIAspectRatioConstraint`
- 底层做完整圆环
- 左右半圆分别控制显示
- 通过旋转 frame 实现进度变化

## 21. Kekui / KUI 插件重点功能

KUI 插件用于加速 Roblox UI 导入。

主要模块：

- Transform
- Quick Actions
- UI Constraints
- UI Gradients

常用能力：

- Offset 转 Scale
- 自动缩放 Position
- 自动缩放 Size
- 缩放 UIStroke
- 缩放 UICorner
- 快速 fit to parent
- 快速 center
- 快速隐藏背景
- 批量处理子元素

## Ultra Scale

视频中重点推荐的功能。

作用：

> 将选中父级下所有 descendant 的 Position、Size、CanvasSize、UIStroke、UICorner 等从 Offset 转为 Scale。

优点：

- 节省大量手动调整时间
- 快速修复不同分辨率下的 UI 变形
- 适合处理复杂导入 UI

## 22. 完整 Anime UI 导入流程

视频最后用一个完整 Anime UI 做实战。

大致步骤：

1. 在设计软件中拆分并导出素材
2. 跳过普通文本
3. 将光效等元素灰度化导出
4. 用小号批量上传图片到 Roblox
5. 创建 `ScreenGui`
6. 设置 `IgnoreGuiInset`
7. 创建主 Frame
8. 放入背景、header、holder 等 ImageLabel
9. 用 ZIndex 管理遮挡关系
10. 创建滚动区域
11. 用 `UIGridLayout` 排列卡片
12. 设置 `AutomaticCanvasSize`
13. 创建卡片内部结构
14. 用 ImageLabel 放角色、稀有度、特效、图标
15. 用 TextLabel 重建等级、名称、状态等文本
16. 给按钮添加 ImageButton 或 hitbox
17. 调整按钮点击范围
18. 用 UIStroke / UIGradient / UICorner 还原视觉细节
19. 用 Device Emulator 检查适配
20. 用 Ultra Scale 统一缩放

## 23. 总结原则

- UI 不应整张图导入，而应拆成可维护模块
- 文本尽量在 Studio 中重建
- PNG 是 UI 导入首选格式
- 图片尺寸不要超过 Roblox 的实际需求
- 用小号上传素材更稳妥
- 层级结构决定 UI 是否好维护
- Scale 比 Offset 更适合跨设备适配
- 不要滥用 AspectRatioConstraint
- ScrollingFrame、UIGridLayout、UIListLayout 是复杂 UI 的核心
- 灰度导入能让颜色动态可控
- 9-Slice 能避免按钮和边框拉伸变形
- KUI 的 Ultra Scale 是处理复杂 UI 的高效工具
