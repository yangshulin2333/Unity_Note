# P1：HUD 接真实数据

目标：让 HUD 不再显示假数据，而是显示服务端玩家数据。

---

## P1-1：完善 DataService 数据结构

现在你是内存版数据。下一步先继续用内存版，但结构要稳定。

建议数据结构：

```lua
{
	HasSelectedLord = false,
	Faction = nil,
	LordId = nil,

	Level = 1,
	Exp = 0,

	Gold = 100,
	Wood = 0,
	Stone = 0,

	Inventory = {},
	Equipped = {},

	UnlockedDungeons = {
		TestDungeon = true,
	}
}
```

完成标准：

- 新玩家创建默认数据
    
- 选主公后写入 Faction
    
- 服务端能读取 Gold/Wood/Stone/Level
    

---

## P1-2：新增 PlayerDataSync

目标：服务端把玩家数据推给客户端 HUD。

需要新增或使用 Remote：

```text
PlayerDataUpdate
```

流程：

```text
玩家进入
↓
DataService 创建数据
↓
服务端 FireClient(PlayerDataUpdate, data)
↓
客户端 HUDController 更新显示
```

完成标准：

- HUD 显示服务端真实数据
    
- 不再依赖假数据
    
- 选主公后可以同步 Faction/Level 等
    

---

## P1-3：HUDController 改为接收数据更新

现在：

```text
HUDController 直接写假数据
```

改成：

```text
HUDController 监听 PlayerDataUpdate
收到数据后刷新 Gold/Wood/Stone/Level
```

完成标准：

- 服务端数据变了，HUD 会更新
    
- 以后采集、奖励、购买都可以统一刷新 HUD

# P3：资源采集系统

目标：资源区真的能产出木材和石头，并刷新 HUD。

这是你前面准备做但暂停的功能。建议在 UI 框架完成后继续。

---

## P3-1：资源点命名规范

场景同事需要按规则放：

```text
Workspace
└─ Interactables
   └─ ResourceNodes
      ├─ Trees
      │  ├─ Tree_01
      │  ├─ Tree_02
      │  └─ Tree_03
      └─ Rocks
         ├─ Rock_01
         ├─ Rock_02
         └─ Rock_03
```

每个资源点建议是：

```text
Model 或 Part
PrimaryPart / BasePart
Attribute:
ResourceType = "Tree" 或 "Rock"
```

---

## P3-2：采集提示 UI

类似传送提示，但内容不同：


可以复用 `TeleportPromptGui` 的思路，也可以做：

```text

```

完成标准：

- 靠近资源点显示提示
    
- 离开隐藏

# 推荐优先级排序

我建议你现在按这个顺序选：

## 第一优先级：P0

**主界面 UI 框架 + UIManager + MainMenuController**

原因：  
你现在已经说要做主界面 UI，而且这是你后续所有功能入口。

---

## 第二优先级：P1

**HUD 接真实数据**

原因：  
后面采集、商店、战斗都要刷新资源，没有真实数据同步会很乱。

---

## 第三优先级：P3

**资源采集系统**

原因：  
资源是经济循环的起点，也能和场景同事资源点联调。

---

## 第四优先级：P4 + P5

**商店 + 背包**

原因：  
有资源以后，需要消耗和物品反馈。

---

## 第五优先级：P6 + P7

**副本 + 战斗**

原因：  
这是核心玩法，但依赖数据、HUD、奖励、背包，建议别太早做复杂。

---

# 我给你的建议路线

最稳的路线是：

```text
1. MainHUD UI框架
2. UIManager
3. MainMenuController
4. HUD接真实数据
5. 资源采集
6. 商店购买
7. 背包显示

9. 最小战斗
10. 奖励与升级
11. 装备强化

```

# 当前你最该确定的优先级问题

你现在只需要决定下一步走哪条：

## 选择 A：继续 UI

做：

```text
UIManager + MainMenuController + 弹窗打开关闭
```

适合你现在要集中做主界面 UI。

## 选择 B：先接数据

做：

```text
PlayerDataUpdate + HUD真实数据
```

适合你想让已有 UI 先接真实服务端数据。

## 选择 C：做资源采集

做：

```text
资源点 + 采集UI + 服务端加资源
```

适合你要和资源区场景联调。

我建议你选 **A，然后 B，然后 C**。
