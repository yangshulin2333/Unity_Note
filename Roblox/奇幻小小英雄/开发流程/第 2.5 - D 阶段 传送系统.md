# 传送系统（`TeleportController` + 场景点位联调）

这一阶段的目标非常明确：

> 玩家靠近传送点时，能看到提示；按键后，能通过服务端传送到目标区域。

这一步是你和场景同事联调的关键节点，因为后面他只要把点位按规范摆好，你这边就能接交互。

---

# 先说这一步做完后会实现什么

我们先做一个**最小可用版**，不要一上来搞复杂 UI。

## 本阶段完成效果

1. 地图里有传送点 Part
    
2. 玩家靠近某个传送点
    
3. 屏幕出现提示，例如：
    
    - `按 E 前往资源区`
        
    - `按 E 进入副本`
        
4. 玩家按 `E`
    
5. 客户端发 `TeleportRequest`
    
6. 服务端判断合法后传送玩家
    
7. 玩家到达目标位置
    

---

我给你拆成 5 小步：

## 第 1 步

## 在 `StarterGui` 下新建一个 `ScreenGui`


---

## 在 `TeleportPromptGui` 下建结构

```text
TeleportPromptGui
└─ Root
   └─ PromptLabel
```

### 具体对象类型

- `Root`：`Frame`
    
- `PromptLabel`：`TextLabel`
    

---

## 属性建议

### `TeleportPromptGui`

- `Enabled = true`
    
- `ResetOnSpawn = false`
    

### `Root`

- `Visible = false`
    
- `AnchorPoint = 0.5, 1`
    
- `Position = {0.5, 0}, {0.9, 0}`
    
- `Size = {0, 300}, {0, 50}`
    
- `BackgroundTransparency = 0.3`
    

### `PromptLabel`

- `Size = {1,0},{1,0}`
    
- `Text = "按 E 传送"`
    
- `TextScaled = true`
    

---

## 为什么这里用 `Visible = false`

因为提示默认不显示，只有靠近传送点才显示。

---

## 你怎么理解这个配置

它的作用是：

- 每个传送点名字，对应一段提示文字
    
- 每个传送点名字，对应一个目标类型
    

这里的 `Target` 不是直接写坐标，而是写“目标标识”：

- `ResourceSpawn`
    
- `DungeonSpawn`
    
- `FactionSpawn`
    

这样后面服务端再根据这个标识决定传到哪。

# 第 5 步：开始写 `TeleportController`

位置：

```text
StarterPlayerScripts > Client > Controllers > TeleportController
```

把内容替换成下面这个版本：

```lua
local Players = game:GetService("Players")
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local UserInputService = game:GetService("UserInputService")
local RunService = game:GetService("RunService")
local Workspace = game:GetService("Workspace")

local RemoteNames = require(ReplicatedStorage.Shared.Constants.RemoteNames)
local TeleportConfig = require(ReplicatedStorage.Shared.Config.TeleportConfig)

local TeleportController = {}

local initialized = false
local currentTeleporter = nil

local DETECT_DISTANCE = 12

function TeleportController.Init()
	if initialized then
		warn("[TeleportController] 已经初始化过，跳过重复初始化")
		return
	end
	initialized = true

	local player = Players.LocalPlayer
	local playerGui = player:WaitForChild("PlayerGui")

	local promptGui = playerGui:WaitForChild("TeleportPromptGui")
	local root = promptGui:WaitForChild("Root")
	local promptLabel = root:WaitForChild("PromptLabel")

	local remotesFolder = ReplicatedStorage:WaitForChild("Remotes")
	local eventsFolder = remotesFolder:WaitForChild("Events")
	local teleportEvent = eventsFolder:WaitForChild(RemoteNames.Events.TeleportRequest)

	local interactables = Workspace:WaitForChild("Interactables")
	local teleportersFolder = interactables:WaitForChild("Teleporters")

	print("[TeleportController] 初始化完成")

	local function hidePrompt()
		root.Visible = false
		currentTeleporter = nil
	end

	local function showPrompt(teleporterPart)
		local config = TeleportConfig.Teleporters[teleporterPart.Name]
		if not config then
			hidePrompt()
			return
		end

		currentTeleporter = teleporterPart
		promptLabel.Text = config.DisplayText
		root.Visible = true
	end

	RunService.RenderStepped:Connect(function()
		local character = player.Character
		if not character then
			hidePrompt()
			return
		end

		local humanoidRootPart = character:FindFirstChild("HumanoidRootPart")
		if not humanoidRootPart then
			hidePrompt()
			return
		end

		local nearestTeleporter = nil
		local nearestDistance = DETECT_DISTANCE

		for _, teleporterPart in ipairs(teleportersFolder:GetChildren()) do
			if teleporterPart:IsA("BasePart") then
				local distance = (humanoidRootPart.Position - teleporterPart.Position).Magnitude
				if distance <= nearestDistance then
					nearestDistance = distance
					nearestTeleporter = teleporterPart
				end
			end
		end

		if nearestTeleporter then
			showPrompt(nearestTeleporter)
		else
			hidePrompt()
		end
	end)

	UserInputService.InputBegan:Connect(function(input, gameProcessedEvent)
		if gameProcessedEvent then
			return
		end

		if input.KeyCode == Enum.KeyCode.E and currentTeleporter then
			print("[TeleportController] 请求传送:", currentTeleporter.Name)
			teleportEvent:FireServer(currentTeleporter.Name)
		end
	end)
end

return TeleportController
```

---

# 你先理解这个控制器在做什么

它做了三件事：

## 1. 每一帧检查

玩家附近有没有传送点

## 2. 如果有

显示提示 UI，并记住当前最近传送点是谁

## 3. 按 E

把这个传送点名字发给服务端

# 第 7 步：写服务端传送逻辑

为了不把 `SelectionSystem` 搞得越来越杂，我们这一步把“普通传送请求”直接先写到 `WorldTeleportService` 里。

打开：

```text
ServerScriptService > Server > Services > WorldTeleportService
```

替换成下面这个版本：

```lua
local Workspace = game:GetService("Workspace")
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local ServerScriptService = game:GetService("ServerScriptService")

local TeleportConfig = require(ReplicatedStorage.Shared.Config.TeleportConfig)

local ServerFolder = ServerScriptService:WaitForChild("Server")
local ServicesFolder = ServerFolder:WaitForChild("Services")

local DataService = require(ServicesFolder:WaitForChild("DataService"))

local WorldTeleportService = {}

local function teleportCharacterToPart(player, targetPart)
	if not player.Character then
		return false, "角色不存在"
	end

	local humanoidRootPart = player.Character:FindFirstChild("HumanoidRootPart")
	if not humanoidRootPart then
		return false, "HumanoidRootPart 不存在"
	end

	humanoidRootPart.CFrame = targetPart.CFrame + Vector3.new(0, 3, 0)
	return true
end

function WorldTeleportService.GetTargetPart(player, teleporterName)
	local config = TeleportConfig.Teleporters[teleporterName]
	if not config then
		return nil, "找不到传送点配置"
	end

	local mapFolder = Workspace:WaitForChild("Map")

	if config.Target == "ResourceSpawn" then
		local resourceZone = mapFolder:WaitForChild("ResourceZone")
		return resourceZone:WaitForChild("ResourceSpawn")
	end

	if config.Target == "DungeonSpawn" then
		local dungeonInstances = mapFolder:WaitForChild("DungeonInstances")
		return dungeonInstances:WaitForChild("DungeonSpawn")
	end

	if config.Target == "FactionSpawn" then
		local data = DataService.GetPlayerData(player)
		if not data or not data.Faction then
			return nil, "玩家没有阵营数据"
		end

		local spawnPoints = mapFolder:WaitForChild("SpawnPoints")
		local spawnName = data.Faction .. "Spawn"
		local spawnPart = spawnPoints:FindFirstChild(spawnName)

		if not spawnPart then
			return nil, "找不到阵营出生点"
		end

		return spawnPart
	end

	return nil, "未知目标类型"
end

function WorldTeleportService.TeleportByTeleporter(player, teleporterName)
	local targetPart, err = WorldTeleportService.GetTargetPart(player, teleporterName)
	if not targetPart then
		return false, err
	end

	return teleportCharacterToPart(player, targetPart)
end

return WorldTeleportService
```

# 第 8 步：新增服务端 `TeleportSystem`

位置：

```text
ServerScriptService > Server > Systems > TeleportSystem
```

如果没有这个 `ModuleScript`，现在新建一个。

内容写：

```lua
local ReplicatedStorage = game:GetService("ReplicatedStorage")
local ServerScriptService = game:GetService("ServerScriptService")

local RemoteNames = require(ReplicatedStorage.Shared.Constants.RemoteNames)

local ServerFolder = ServerScriptService:WaitForChild("Server")
local ServicesFolder = ServerFolder:WaitForChild("Services")

local WorldTeleportService = require(ServicesFolder:WaitForChild("WorldTeleportService"))

local TeleportSystem = {}

function TeleportSystem.Init()
	local remotesFolder = ReplicatedStorage:WaitForChild("Remotes")
	local eventsFolder = remotesFolder:WaitForChild("Events")

	local teleportEvent = eventsFolder:WaitForChild(RemoteNames.Events.TeleportRequest)

	teleportEvent.OnServerEvent:Connect(function(player, teleporterName)
		print("[TeleportSystem] 收到传送请求:", player.Name, teleporterName)

		if type(teleporterName) ~= "string" then
			warn("[TeleportSystem] teleporterName 不是字符串")
			return
		end

		local success, result = WorldTeleportService.TeleportByTeleporter(player, teleporterName)
		if not success then
			warn("[TeleportSystem] 传送失败:", result)
			return
		end

		print("[TeleportSystem] 传送成功:", player.Name, teleporterName)
	end)

	print("[TeleportSystem] 初始化完成")
end

return TeleportSystem
```

# 第 11 步：这一步最容易出的问题

我先提前帮你列出来，省得你卡住。

## 问题 1：提示 UI 不出现

优先查：

- `TeleportPromptGui` 是否建好
    
- `Root.Visible` 是否默认 false
    
- `TeleportController` 是否被 `ClientBootstrap.Init()`
    
- 传送点是不是 `BasePart`
    

## 问题 2：按 E 没反应

优先查：

- Output 有没有 `[TeleportController] 请求传送`
    
- `currentTeleporter` 有没有被正确记录
    
- `TeleportRequest` 是否存在
    

## 问题 3：服务端收到请求但没传送

优先查：

- `TeleportConfig` 里这个传送点名字是否存在
    
- `ResourceSpawn` / `DungeonSpawn` 是否真的建好了
    
- `FactionSpawn` 逻辑里 `DataService` 是否有阵营数据
    

## 问题 4：传错地方

优先查：

- 传送点名字是否和配置匹配
    
- 场景同事是否改了点位名字
    
- `Target` 配置是否写错
