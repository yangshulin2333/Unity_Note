# 为什么是 ModuleScript？

因为这三个东西不是要“自己一放进去就自动运行”的脚本，
而是要被别的脚本 `require()` 进来使用的“模块”。

你现在先这样理解就行：

- `Script`：放进去会自己运行
- `LocalScript`：客户端自己运行
- `ModuleScript`：不会自己运行，必须被别的脚本加载

---

# 那它们现在的作用是什么？

这三个现在只是**玩法模块占位**。

也就是说，当前阶段你先把它们建成 `ModuleScript`，后面再往里面写逻辑。

# 为什么不是普通 Script？

因为如果你把它们建成普通 `Script`，会有几个问题：

## 1. 它们会自己直接运行
但你当前还没有统一控制它们什么时候启动。

## 2. 初始化顺序会变乱
比如：
- DataService 还没准备好
- SelectionSystem 已经开始跑了

就容易报错。

## 3. 后面不好统一管理
我们现在希望所有系统都由 `ServerBootstrap` 控制启动。

所以：

- `ServerBootstrap` 是总启动器
- `SelectionSystem / ResourceSystem / DungeonSystem` 是模块
- 模块由 `ServerBootstrap` `require` 后启动

这就是为什么它们应该是 `ModuleScript`
