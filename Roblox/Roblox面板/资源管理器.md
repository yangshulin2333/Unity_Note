Roblox 资源管理器主要包含两大核心工具 ——**Explorer（对象资源管理器）** 与 **Asset Manager（素材管理器）**，分别负责项目内对象层级组织与外部素材批量管理，是开发中场景搭建、脚本控制与素材入库的核心入口。

### 一、Explorer（对象资源管理器）

Explorer 以**父子层级树**展示项目所有实例（如零件、脚本、服务等），是场景结构与逻辑组织的核心窗口Roblox。

#### 1. 打开方式

- 视图 → Explorer（默认停靠左侧）Roblox；
- 主页 → 窗口 → ExplorerRoblox。

#### 2. 核心功能与操作

#### 3. 常用服务与容器说明

|服务 / 容器|用途|
|---|---|
|**Workspace**|3D 世界本体，所有可见对象（零件、模型、角色、附件、物理对象）都在这里|
|**Players**|管理所有玩家，包含 `PlayerAdded`、`PlayerRemoving` 事件，存放所有 Player 对象|
|**StarterPlayer**|玩家初始配置，包含子文件夹：`StarterPlayerScripts`、`StarterCharacterScripts`|
|**StarterPlayerScripts**|玩家**加载时自动运行**的客户端脚本（生命周期跟随玩家）|
|**StarterCharacterScripts**|角色**重生时自动运行**的客户端脚本（每次复活都重新执行）|
|**StarterGui**|存放 UI 界面，玩家进入游戏后会被复制到 PlayerGui|
|**StarterPack**|存放工具 / 武器，玩家加入时自动放进背包|
|**ServerScriptService**|服务器脚本专用容器，只在服务器运行，安全、防作弊|
|**ServerStorage**|服务器私有资源库，客户端**无法访问**，适合存放怪物、武器、地图等机密资源|
|**ReplicatedStorage**|公共资源库，服务器和客户端都能访问，用于存放共享模型、RemoteEvent、模块脚本|
|**ReplicatedFirst**|优先加载的资源，常用于加载界面、预加载脚本，保证最先执行|
|**Lighting**|全局光照、天空盒、时间、阴影、环境效果、后处理等|
|**SoundService**|全局音效管理、混响、音量、空间音效设置|
|**Chat**|聊天系统，管理游戏文字聊天、气泡消息|
|**Teams**|队伍系统，用于分组、阵营、PVP 敌我区分|
|**TextChatService**|新一代聊天系统，替代旧 Chat，更灵活可控|
|**RunService**|核心循环服务：`Stepped`、`Heartbeat`、`RenderStepped`，驱动游戏每帧逻辑|
|**TweenService**|平滑动画服务，用于移动、旋转、透明度、颜色等缓动动画|
|**CollectionService**|标签管理，给对象打标签，批量获取同类对象|
|**UserInputService**|客户端输入服务：键盘、鼠标、触屏、游戏手柄检测|
|**Camera**|相机对象（属于 Workspace），控制视角、FOV、镜头效果|
|**TeleportService**|跨地图 / 跨服务器传送服务|
|**DataStoreService**|数据存储服务，用于保存玩家金币、等级、存档等（服务器专用）|
|**PhysicsService**|物理碰撞组管理，设置哪些物件能碰撞、哪些穿透|
|**MarketplaceService**|游戏 pass、道具、付费产品、开发者产品购买处理|
|**BadgeService**|成就徽章服务，授予、检查玩家徽章|
|**HttpService**|网络请求服务，GET/POST、JSON 解析（需开启权限）|

#### 超简记忆口诀（方便记）

- **看得见的 → Workspace**
- **玩家相关 → Players / StarterPlayer**
- **UI → StarterGui**
- **工具 → StarterPack**

- **服务器私藏资源 → ServerStorage**

### 二、Asset Manager（素材管理器）

Asset Manager 负责**外部素材批量导入、入库与管理**，支持图片、网格、音频、模型等多类型素材，是资源入库与复用的关键工具Roblox。

#### 1. 打开方式

- 主页 → 素材管理器Roblox；
- 窗口 → 素材管理器Roblox。

#### 2. 核心功能与操作

- 批量导入：单次最多 50 个文件，自动进入审核队列，仅创作者可见Roblox；
- 素材类型与格式：
    
    - 图片：PNG/JPG/GIF/TGA/BMP（用于纹理、贴花）Roblox；
    - 网格：FBX/OBJ（复杂骨骼 / 动画建议用 3D Importer）Roblox；
    - 音频：OGG/MP3/FLAC/WAV（需合法授权， Creator Store 有免费素材）Roblox；
    - 视频：MP4/MOVRoblox；
    
- 素材操作：右键可重命名、复制 ID、预览、从项目移除Roblox；
- 视图切换：网格 / 列表视图，支持筛选与排序Roblox。

### 三、两者协作与开发流程

1. **素材入库**：用 Asset Manager 导入外部素材，获取素材 IDRoblox；
2. **场景搭建**：在 Explorer 中组织 Workspace 内 3D 内容，插入零件、模型与脚本Roblox；
3. **资源关联**：将素材 ID 赋值给对应对象（如 MeshPart.MeshId、Sound.SoundId）Roblox；
4. **逻辑控制**：在 ServerScriptService/StarterPlayerScripts 中编写脚本，通过 Explorer 层级访问对象Roblox；
5. **测试与发布**：在 Studio 中测试，确认素材与对象层级无误后发布。

### 四、实用技巧与注意事项

1. **组织规范**：用 Folder 分类管理（如 “环境音效”“角色模型”），保持层级清晰Roblox；
2. **搜索技巧**：
    
    - 类检索：is:Part（所有零件）、is:BasePart（所有基础零件）Roblox；
    - 属性检索：Locked=true（锁定的零件）、Material==Plastic（塑料材质）Roblox；
    - 路径检索：Workspace.Model.Child（指定路径对象）Roblox；
    
3. **素材审核**：导入后需通过官方审核，违规素材可能导致账号限制Roblox；
4. **批量操作**：多选对象（Ctrl/Shift + 点击）进行批量复制、移动或删除Roblox。

### 五、常见问题

- Explorer 消失：视图 → Explorer 重新打开，或重置窗口布局Roblox；
- 素材无法显示：检查素材 ID 是否正确，或是否通过审核Roblox；
- 层级混乱：用 Folder 归类，合理设置父级关系，避免深层嵌套Roblox。
