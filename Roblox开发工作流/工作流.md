# 🚀 Roblox 专业开发工作流配置指南

**(VS Code + Rojo + Wally + Git)**

## 第一阶段：核心基础环境搭建

### 1. 安装核心工具

1. **下载并安装 VS Code:** 现代代码编辑器的行业标准。
    
2. **下载并安装 Rokit:** * **作用：** 这是一个工具链管理器，用来统一安装和管理 Roblox 的各种开发命令行工具（如 Rojo, Wally）。
3. [下载链接](https://github.com/rojo-rbx/rokit)
    
    - **操作：** 去 GitHub 搜索 `Rokit roblox`，下载最新版 `.exe` 并运行（如遇 Windows 安全提示，选择“仍然运行”）。**安装后必须重启电脑**。
        

### 2. 创建你的项目空间

1. 在电脑的文档目录中创建一个总文件夹，例如 `RobloxProjects`。
    
2. 在里面为你的游戏创建一个专属文件夹，例如 `HengSaoQianJun`。
    
3. **在 VS Code 中打开这个文件夹** (`File -> Open Folder`)。
    

---

## 第二阶段：打通编辑器与引擎 (Rojo)

**Rojo** 是这套工作流的心脏，它负责把你能在 VS Code 里写的代码，实时且毫无延迟地同步到 Roblox Studio 里面去。

### 1. 安装 VS Code 扩展 (Extensions)

在 VS Code 左侧的扩展商店，搜索并安装以下两个核心插件：

- **Luau Language Server:** 提供 Roblox 专属的 Luau 语言支持（语法高亮、代码提示）。
    
- **Rojo:** Rojo 的 VS Code 客户端。
    

### 2. 初始化 Rojo 项目

在 VS Code 中按下 `` Ctrl + ` `` 打开终端 (Terminal)，依次输入以下命令：

Bash

```
rokit init
rokit add rojo
rojo init
```

_执行完毕后，你的文件夹里会多出几个文件，最重要的是 `default.project.json` 和 `src` 文件夹。_

### 3. 配置项目映射 (default.project.json)

打开 `default.project.json`，这里定义了本地文件如何对应到 Studio。

- **建议：** 删除 JSON 中 `Workspace` 相关的部分（保留 `ReplicatedStorage` 等），因为你的地图、模型（比如关羽）依然在 Studio 里做，VS Code 只负责写代码。
    

### 4. 连接到 Roblox Studio

1. 在 VS Code 按 `Ctrl + Shift + P`，输入 `Rojo: Start server`（或点击底部状态栏的 Rojo 图标）。
    
2. 打开 Roblox Studio，进入你的游戏地图。
    
3. 在 Studio 的顶部菜单找到 **Rojo 插件**，点击 **Connect**。
    
    _此时，你在 VS Code `src` 文件夹里写的代码，会自动同步到 Studio 里！_

## 第三阶段：代码规范与自动化 (StyLua & Selene)

专业团队的代码看起来就像一个人写出来的，这归功于格式化工具和静态分析工具。

### 1. 安装 VS Code 扩展

- **StyLua:** 自动格式化代码（帮你调整缩进、空格）。
    
- **Selene:** Linter，自动检查代码里的坏习惯或潜在 Bug。
    

### 2. 配置自动格式化

1. 在 VS Code 按 `Ctrl + Shift + P`，搜索 `Preferences: Open User Settings (JSON)`。
    
2. 添加配置，让每次保存时自动格式化 Luau 代码。
    
3. **设置 Luau 自动导包 (Auto-imports):** 在扩展设置中开启 `Luau -> Imports` 功能，以后输入服务名（如 `ReplicatedStorage`）会自动帮你补全 `game:GetService(...)`。
    

### 3. 配置 Selene

在项目根目录新建一个文件 `selene.toml`，输入一行基础配置：

Ini, TOML

```
std = "roblox"
```

## 第四阶段：包管理器配置 (Wally) —— 重点！

**Wally** 就是 Roblox 界的 NuGet。你之前苦恼“不知道去哪找 Matter 的官方代码”，有了 Wally，只需要写一行配置，它自动帮你下载好。

### 1. 安装 Wally

在 VS Code 终端依次输入：

Bash

```
rokit add wally
rokit add wally-package-types
wally init
```

_这会生成一个 `wally.toml` 文件（类似 C# 的 .csproj 里的 PackageReferences）。_

### 2. 引入开源库 (以 Matter 和 ProfileService 为例)

打开 `wally.toml`，你会看到 `[dependencies]` 区域。你可以去 wally.run 网站搜索你想要的库。

修改你的配置如下：

Ini, TOML

```
[dependencies]
Matter = "evaera/matter@0.1.4"
Fusion = "elttob/fusion@0.3.0"

[server-dependencies]
ProfileService = "madstudiosdev/profileservice@1.4.2"
```

### 3. 下载并同步包

在终端输入：

Bash

```
wally install
```

_Wally 会自动生成 `Packages` 和 `ServerPackages` 文件夹，里面装满了下载好的源码！_

### 4. 让 Rojo 识别这些包

打开 `default.project.json`，将刚刚生成的包文件夹映射进去：

- 将 `"Packages": { "$path": "Packages" }` 添加到 `ReplicatedStorage` 下。
    
- 将 `"ServerPackages": { "$path": "ServerPackages" }` 添加到 `ServerScriptService` 下。

## 第五阶段：版本控制 (Git & GitHub)

没有版本控制的开发无异于走钢丝。Git 让你拥有无数次“撤销”到任意一天的能力。

### 1. 基础配置

1. 安装 Git 客户端，并创建一个 GitHub 账号。
    
2. 在 VS Code 左侧的 **Source Control（源代码管理）** 界面点击 `Initialize Repository`。
    

### 2. 配置 .gitignore (非常重要)

你不需要把 Wally 下载的开源包也传到网上去。新建一个 `.gitignore` 文件，添加：

Plaintext

```
Packages/
ServerPackages/
sourcemap.json
wally.lock
```

### 3. 提交与分支 (Commit & Branch)

- **Commit:** 每次写完一个核心功能（比如写完了 `HeroSpawnSystem`），写一句描述，点击 Commit 存档。
    
- **Branch:** 如果你要开发一个新功能（比如加入战马系统），但不确定会不会弄崩现有的代码，可以新建一个 Branch (分支)。做坏了直接删分支，做好了再 Merge (合并) 进主代码中。

## 💡 给 HIS 转型开发者的总结笔记

这套工作流配完后，你的开发体验将发生**质的飞跃**：

1. **写代码就像写 C# 项目：** 所有的业务逻辑都在 VS Code 里写，结构清晰。文件后缀有明确意义（`.server.luau` 是只在服务端运行，`.client.luau` 是客户端运行）。
    
2. **再也不用去工具箱 (Toolbox) 淘代码：** 需要什么框架，直接去 Wally 搜，改一下 `wally.toml` 就能用，干净卫生无病毒。
    
3. **Roblox Studio 降级为“场景编辑器”：** 你以后打开 Studio，只做三件事：**摆放 UI、调整地图模型、点击 Play 运行测试**。所有的代码工作都在 VS Code 里完成。
