
## **流程图生成 Prompt 规则**

```
生成一个 Mermaid 流程图，用于 Obsidian 渲染，要求如下：

1. 节点规则
   - 每个模块作为一个父节点，每个父节点下可以有若干子节点，显示模块的详细功能。
   - 每条功能用斜杠 "/" 或斜线分隔。
   - 父节点表示模块名称，子节点表示模块具体功能。
   - 节点内容尽量保持一行，不使用 \n 或 <br>，保证 Obsidian 渲染不报错。

2. 流程关系
   - 用箭头 "-->" 表示流程方向。
   - 循环逻辑用箭头标注，例如战斗完成返回大厅：`Battle -->|战斗完成| Lobby`。
   - 子节点连接到父节点，父节点之间按逻辑顺序连接。

3. 模块示例（可替换成你的模块和功能）
   - 模块1: 功能1 / 功能2 / 功能3
   - 模块2: 功能1 / 功能2
   - 模块3: 功能1 / 功能2 / 功能3 / 功能4
   ...
   - 流程关系：
     模块1 --> 模块2
     模块2 --> 模块3
     模块3 --> 模块1 (如果需要循环)

4. 可选高级规则
   - 可以用 classDef 给模块类型标颜色，例如逻辑 / 数据 / UI / FX。
   - 每个模块的父节点与子节点用箭头连接，保持详细信息完整。
   - 循环箭头清晰标注。
   - 输出 Mermaid flowchart 语法，可直接复制到 Obsidian 渲染。

请根据以上规则生成完整 Mermaid 流程图，逻辑顺序正确，父节点 + 子节点详细功能齐全，循环箭头表示功能回环。
```
